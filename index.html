shurl

URL shortener

This URL shortener is unlike the others in that there is no database, so there isn't much to break or break into. It should last longer and require less work.

The front end is a client-side console app which does the work of adding short URLs to the .htaccess file on your web server.

Under the hood, the trick is that the Apache mod_rewrite file is used to store rewrites. The privacy of your shortened URLs is protected by the fact that short URLs in this system are random strings, not serial numbers, so third parties can't spider them.

The entire system is contained in a single shell script, shurl. On first run it will create a configuration file. Edit the file to provide:
  1. The public URL at which your shortener will be hosted
  2. SSH address for your web server
  3. Path to .htaccess in your web server's file system

From that point on, shorten a url like this:
shurl [LONG URL TO SHORTEN].

This is designed as a client-server system where the URL is added on a client machine, then propagates to a server machine. Would it be useful to have a mode for operating within a single machine, so that the command to add a URL is invoked in a shell on the web server? Comments welcome.
